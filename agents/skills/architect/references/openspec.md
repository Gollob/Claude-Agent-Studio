# OpenSpec — spec-driven development

Источник: <https://github.com/Fission-AI/OpenSpec> (Fission-AI/OpenSpec). Лёгкий слой спек поверх AI-кодинга: договариваемся, ЧТО строим, до написания кода. Без API-ключей, работает с 20+ ассистентами через slash-команды.

## Установка / инициализация
```bash
# глобально (если разрешена запись в npm-prefix; иначе через npx — без установки):
npm install -g @fission-ai/openspec@latest

# в проекте — инициализация (создаёт каталог openspec/ и команды):
openspec init
#   через npx без глобальной установки:
npx -y @fission-ai/openspec@latest init
```

## Структура каталога
```
openspec/
  changes/
    <change-id>/
      proposal.md   # зачем и что меняем (мотивация, scope)
      specs/        # требования и сценарии (что система должна делать)
      design.md     # технический подход: компоненты, ADR, диаграммы
      tasks.md      # чек-лист реализации (наряд на делегирование)
  specs/            # «живые» спеки после archive (накопленное знание о системе)
```

## Рабочий цикл
1. **Propose** — `/opsx:propose <что строим>` (или вручную создать `changes/<id>/` и заполнить 4 артефакта). Человек и агент выравниваются на «что» и «как».
2. **Apply** — `/opsx:apply` — реализация задач из `tasks.md` точечными изменениями. В студии это делают dev-агенты по `tasks.md`.
3. **Archive** — `/opsx:archive` — завершённое изменение переносится в «живые» спеки `openspec/specs/`.

Расширенный профиль команд (`openspec config profile` → `openspec update`): `/opsx:new`, `/opsx:continue`, `/opsx:ff`, `/opsx:verify`, `/opsx:bulk-archive`, `/opsx:onboard`.

## Принцип для архитектора студии
- Никаких жёстких фазовых гейтов: любой артефакт можно править в любой момент, но **код не начинаем, пока `proposal.md` + `specs/` не согласованы**.
- `tasks.md` — это контракт с дирижёром: атомарные задачи, помеченные профильным агентом, с критерием готовности.
- Если slash-команды OpenSpec недоступны в текущем агенте — архитектор **сам** создаёт и заполняет файлы по этой структуре; `npx openspec validate` (при наличии) проверяет корректность.
